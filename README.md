# Perturbation Evaluation Audit — Notebooks

Reproducibility notebooks for:

> **Single-shot perturbation benchmarks can mislead: diagnostics for split- and metric-driven instability in single-cell model evaluation**
> Whalley, J.P. (2026). Submitted to MLCB 2026.

## Overview

This repository contains the notebooks needed to reproduce every figure, table, and headline statistic in the manuscript. The default reproducibility path is intentionally lightweight: clone the repository, install dependencies, and run the figure notebooks against the shipped `precomputed/` CSVs in minutes. Full retraining (approximately 13 hours of CPA plus matched-n GEARS) is optional.

```
D01–D03  →  P01–P04  →  01–04
(acquire)   (analyse)   (visualise)
```

Three layers:

1. **Data acquisition (D01–D03).** Download Replogle atlases, build per-gene severity references, generate holdout specifications.
2. **Analysis pipeline (P01–P04).** Train CPA and GEARS, evaluate severity correlations, compute LOO sensitivity, run the winsorisation panel.
3. **Figure notebooks (01–04).** Consume CSVs and produce Figure 1, Tables 1 and 2, and Appendix A.

## Quick start (figures only, from precomputed CSVs)

```bash
git clone <repo-url>
cd repository/
uv sync
./run_all.sh --figures
```

Approximate runtime: 2 minutes.

## Run modes

```bash
./run_all.sh                  # print help (does not start heavy compute)
./run_all.sh --figures        # figure/table notebooks from precomputed CSVs (~2 min)
./run_all.sh --demo           # 2-seed pipeline run to verify installation (~30 min)
./run_all.sh --analysis       # recompute eval CSVs from existing per-seed predictions
./run_all.sh --train-full     # full n=100 CPA + matched-n GEARS (~13 hours)
```

Heavy modes (`--demo`, `--train-full`) require the model dependency groups:

```bash
uv sync --group cpa --group gears
./run_all.sh --train-full
```

## Prerequisites

- Python 3.11
- [uv](https://docs.astral.sh/uv/) for dependency management

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

For full training only:

- Apple Silicon (MPS) or CUDA GPU recommended
- ~20 GB disk space for atlases, predictions, and logs

## Notebook execution status

`run_all.sh` skips notebooks that have not yet been marked ready in the `NOTEBOOKS_READY` list inside the script. This prevents the scaffold from appearing runnable before the implementations are in place. As each notebook is implemented and tested, it is added to that list.
