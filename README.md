# Perturbation Evaluation Audit — Notebooks

Reproducibility notebooks for:

> **Diagnosing split sensitivity in severity-correlation evaluation for single-cell perturbation prediction**
> Whalley, J.P. (2026). Submitted to MLCB 2026.

## Default path: reproduce all manuscript figures and tables in two minutes

```bash
git clone https://github.com/jpwhalley/perturb-eval-notebooks
cd perturb-eval-notebooks
uv sync
./run_all.sh --figures
```

This reproduces **Figure 1, Table 1, Table 2, Appendix A, and Appendix B** (and the §3.3 Spearman / metric-robustness numbers) from the shipped CSVs in `precomputed/`. No external data, no model training, no GPU. Approximate runtime: 2 minutes on a laptop.

Everything beyond this point is optional.

## Architecture

The repository has three layers, each with a defined role. Understanding the separation matters because the layers correspond to different reproducibility commitments.

| Layer | Role | Reproducibility commitment |
|---|---|---|
| **`precomputed/`** | Canonical manuscript layer | Byte-for-byte reproduction of every manuscript value. Tracked in git; never regenerated by the public-default path. |
| **`scripts/`** | Implementation (CLI) | Validated reproduction of the precomputed layer from upstream artefacts. Each script has a `--validate` mode that diffs its output against `precomputed/`. |
| **Notebooks (01–05, P01–P04, D01–D03)** | Exposition and verification | Readable accounts of what each step does. The figure/table notebooks (01–05) regenerate `precomputed/` outputs from the shipped CSVs; the P-series notebooks call `scripts/` for the heavy operations. |

**Data flow:**

```
raw atlases  →  raw predictions  →  severity-detail h5ads  →  evaluation CSVs  →  figures + tables
              (P01, P02 training)   (scripts/evaluate_         (scripts/            (01–05 notebooks
                                      severity_panel.py)        recompute_            consuming
                                                                 diagnostics.py)      precomputed/)
                                                                                       01-05 notebooks
```

Each arrow is a documented, testable transformation. The right three arrows are pure data transformations and are end-to-end validated against the manuscript-canonical artefacts. The leftmost arrow (raw atlases → raw predictions) is the only step requiring model training; it remains documented as a separate engineering layer (see "Reproducibility levels" below).

## Reproducibility levels

Four increasingly demanding levels of reproduction:

| Level | Command | Time | Implementation | Inputs you need |
|---|---|---|---|---|
| **1** | `./run_all.sh --figures` | ~2 min | Notebooks 01–05 read shipped CSVs | Cloned repo + `uv sync` |
| **2** | `python3 scripts/recompute_diagnostics.py --validate` | ~30 s | Phase 1 CLI (validated against `precomputed/`) | Per-seed severity-detail h5ads in `data/predictions/severity_details/` |
| **2.5** | `python3 scripts/evaluate_severity_panel.py` | ~minutes | Phase 2 CLI (validated against existing severity details) | Raw prediction h5ads in `data/predictions/raw/` + atlases in `data/replogle/` |
| **3** | `./run_all.sh --train-full` | ~13 hours | Documented engineering layer; not yet implemented as standalone CLIs | All of the above + CPA and GEARS installed + raw Replogle atlases |

Level 1 is the public-default path, what a reviewer or casual reader will use. Level 2 confirms that the diagnostic transformations (LOO sensitivity, winsorisation, mechanism summary, etc.) faithfully recompute from severity-detail inputs. Level 2.5 confirms that the severity-detail h5ads themselves are faithfully derived from raw model predictions. Level 3, the actual training, remains scoped as a separate session because it requires model installations and many hours of compute.

## Run modes

```bash
./run_all.sh                  # print help; does not start any compute
./run_all.sh --figures        # Level 1: public default
./run_all.sh --demo           # training-path smoke test (checks prereqs; trains only if all present)
./run_all.sh --analysis       # recompute eval CSVs from existing predictions
./run_all.sh --train-full     # Level 3: full retraining (currently documented stub, see below)
```

**Mode semantics:**

- **`--figures`** — runs notebooks 01–05 against shipped `precomputed/` CSVs. The 2-minute default reader path.
- **`--demo`** — runs the full notebook chain (D-series → P-series → figures) with `QUICK_DEMO=1` set. Each notebook checks its prerequisites and exits gracefully with installation guidance if anything is missing. To actually run a 2-seed CPA training demo you must have completed `uv sync --group cpa --group gears --group data`, downloaded the Replogle supplementary Excel for D02, and successfully run D01 to populate `data/replogle/`. The smoke test surfaces what is missing rather than performing these steps.
- **`--analysis`** — runs P-series and figure notebooks. P03/P04 fall through gracefully if `data/predictions/severity_details/` is empty.
- **`--train-full`** — same chain as `--demo` with `FULL_TRAINING=1`. Currently exits gracefully at the P01/P02 stubs; full implementation is the documented engineering layer below.

## Prerequisites

- Python 3.11
- [uv](https://docs.astral.sh/uv/) for dependency management

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Level 1 requires only the base dependency group (installed automatically by `uv sync`). Levels 2.5 and 3 additionally require:

```bash
uv sync --group cpa --group gears --group data
```

For Level 3 (full retraining): Apple Silicon (MPS) or CUDA GPU recommended; ~20 GB disk space for atlases, predictions, and logs.

## Status

Implementation status by layer:

- **Layer 1 (`precomputed/`)**: complete. All manuscript-facing artefacts (Figure 1, Tables 1 and 2, Appendix A, Appendix B, bootstrap CIs, and the §3.3 Spearman / metric-robustness numbers) are reproduced byte-for-byte by `./run_all.sh --figures`.
- **Layer 2 (`scripts/`)**: Phases 1 and 2 complete. `scripts/recompute_diagnostics.py` reproduces all 14 manuscript-canonical CSVs in `precomputed/eval/`, `precomputed/tables/`, and `precomputed/figure_inputs/` from severity-detail h5ads, validated against the precomputed reference (14/14 outputs match). `scripts/evaluate_severity_panel.py` reproduces severity-detail h5ads from raw prediction artefacts, validated against 428 existing detail files with zero failures. Phases 3 and 4 (CPA and GEARS training CLIs) remain documented as the next engineering layer.
- **Layer 3 (notebooks)**: 12 notebooks implemented and in `NOTEBOOKS_READY`. Figure/table notebooks consume `precomputed/`; P-series notebooks expose the script-call paths; D-series notebooks document the data acquisition flow with graceful prerequisite checks.

The substantive gap remaining is the training-script implementation (Phases 3 and 4), required only for Level 3 reproduction. It does not affect Levels 1, 2, or 2.5.
